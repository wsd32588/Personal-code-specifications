# Claude Code 提词模板

## 使用方式

把模板复制给 ChatGPT / Gemini 等 AI，让它填充后，把结果直接粘贴给我。

---

## 模板一：提需求 / 修 Bug

```
请帮我生成一份给 Claude Code 的任务描述。

项目背景：[一句话说明项目是什么，技术栈]
任务目标：[要做什么 / 修什么]

输出格式：
1. 任务描述 — 清晰的一句话
2. 具体改动 — 涉及的文件和改动要点
3. 约束条件 — 不允许改的部分
4. 验收标准 — 怎样算改完了

只输出任务描述，不要给我代码。
```

---

## 模板二：代码审查

```
请审查以下代码，列出所有 Bug 和问题。

项目语言：C
关注点：
- 内存安全
- 越界访问
- 未定义行为
- 资源泄漏
- 逻辑错误

输出格式（按优先级）：
1. 🔴 严重 Bug（崩溃 / 数据丢失）
2. 🟠 逻辑缺陷（功能不符合预期）
3. 🟡 代码质量（可读性 / 维护性）

只列问题，不要给修复代码。
```

---

## 模板三：重构 / 迁移

```
请帮我生成一份给 Claude Code 的重构说明。

项目：C 语言文本编辑器
重构目标：[简短描述，如"把全局变量改成参数传递"]

输出要求：
1. 改动清单 — 每个文件改什么
2. 不碰的部分（业务逻辑不变）
3. 验收标准（编译通过、行为不变）

格式：简洁条目化，不要啰嗦。
```

---

## 通用原则

无论用哪个模板，末尾加这句效果最好：

```
**只输出任务描述，不要写代码。Claude Code 会自己实现。**
```

这样 AI 不会自作主张生成代码，而是产出清晰的规格说明，你直接粘贴给我就行。
